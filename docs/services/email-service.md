# Email Service

The Email Service sends automated Gmail reminders to students about pending assignments. It reads message request CSV files (generated by the GradeSync service) and sends personalized emails using the Gmail API.

## Features

- **Automated Sending**: Reads CSV files and sends emails without manual intervention.
- **Personalized Content**: Dynamically inserts student names, assignment details, and resource links.
- **Resource Integration**: Fetches assignment resources from Supabase if not provided in the CSV.
- **Robust Error Handling**: Skips invalid entries and logs errors without stopping the batch.

## Configuration

Required environment variables in `.env.local`:

```bash
# Google Cloud Service Account
# Path is relative to project root or absolute
# Defaults to services/config/oauth_client_secret.json
GOOGLE_APPLICATION_CREDENTIALS="services/config/oauth_client_secret.json"

# Sender Email (must be authorized in Google Workspace)
GMAIL_SENDER_EMAIL="autoremind@yourdomain.com"
```

### Credentials Setup

There are two ways to authenticate with Gmail:

1.  **Service Account (Recommended for Automation)**
    *   Requires a Google Workspace account.
    *   Enable Domain-Wide Delegation for the service account.
    *   Grant the `https://www.googleapis.com/auth/gmail.send` scope.
    *   Place the JSON key file at `services/config/oauth_client_secret.json`.

2.  **OAuth 2.0 (User Credentials)**
    *   Useful for testing with personal Gmail accounts.
    *   Requires a one-time interactive login to generate a refresh token.
    *   Token is stored in `services/config/token.json`.

## Usage

Run the service from the project root:

```bash
python3 services/email-service/main.py
```

### Command Line Arguments

- `--dir`: Directory containing message request CSV files (default: `services/gradesync_input/message_requests`).
- `--file`: Process a specific CSV file.
- `--credentials`: Path to custom credentials file.
- `--sender`: Override the sender email address.
- `--generate`: Run the database fetch step (`db_fetch.py`) before sending.

## Automation

To run this service automatically (e.g., daily), set up a cron job:

```bash
0 9 * * * cd /path/to/remind && python3 services/email-service/main.py --generate
```
